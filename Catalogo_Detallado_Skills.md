## 🏗️ La Fórmula Exacta para Activar una Skill

Para obtener resultados de calidad "Senior", tus peticiones (prompts) deben ser específicas. Usa esta estructura:

> **[Rol/Activación]** + **[Contexto/Archivo]** + **[Problema/Tarea]** + **[Condiciones]**

**Ejemplo perfecto:**
*"Usa la skill **`ui-ux-pro-max`** en el archivo **`src/pages/NuevoRegistro.jsx`**. Haz que el diseño sea tipo 'bento grid' usando Tailwind CSS, **asegúrate** de incluir animaciones sutiles y soporte para Dark Mode."*

## 1. Frontend, UI/UX y Diseño Web

Estas skills transforman a la IA en un Diseñador UX/UI experto y en un Ingeniero Frontend moderno.

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`ui-ux-pro-max`** | Cuando necesitas un diseño visual impactante, moderno y profesional. Domina animaciones, dark mode, glassmorphism, claymorphism y bento grids. | *"Usa `ui-ux-pro-max` para rediseñar la vista del Dashboard. Aplica modo oscuro y estética 'minimalista'."* |
| **`react-nextjs-development`**| Para escribir lógica de React/Next.js impecable, usando los últimos patrones (Server Components, optimización de hooks). | *"Modo `react-nextjs-development`: Necesito programar el filtrado complejo para la tabla de Reportes Globales en React."* |
| **`tailwind-design-system`** | Para sistemas de diseño escalables. Evita tener clases de CSS repetitivas en toda la app estructurando variables madre. | *"Aplica `tailwind-design-system` a mi `index.css` y crea variables lógicas para colores primarios y secundarios."* |
| **`wcag-audit-patterns`** | Auditorías de accesibilidad visual (contrastes) y soporte para navegación con teclado y lectores de pantalla. | *"Corre `wcag-audit-patterns` sobre el menú lateral y verifica que los botones sean accesibles por tabulación."* |
| **`scroll-experience`** | Para generar experiencias interactivas y cinemáticas basadas en el *scroll* del usuario (efectos parallax, storytelling). | *"Crea un Landing Page y usa `scroll-experience` para animar las tarjetas según hago scroll hacia abajo."* |

## 2. Arquitectura, Patrones y Diseño de Sistemas

Usa esto ANTES de codificar, para planificar cómo se comunicarán los pedazos grandes de tu app.

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`architect-review`** | Al introducir una función grande. Busca la mejor herramienta, evalúa *trade-offs* (pros y contras) y patrones de arquitectura. | *"Actúa como `architect-review`. ¿Deberíamos usar Context API o Zustand para manejar las variables en esta App de Auditorías?"* |
| **`domain-driven-design`** | Para alinear el código con la "lógica de negocio" real (lenguaje ubicuo, separación en módulos por dominios de auditoría). | *"Usa `domain-driven-design` y mapea los requerimientos de 'Correlativos de Notas' a modelos de datos."* |
| **`api-design-principles`** | Diseño profesional de llamadas y URLs REST antes de tirar código. | *"Modo `api-design-principles`: Diseña el JSON que el frontend debería enviarle al backend al crear un usuario nuevo."* |
| **`c4-architecture`** | Creación técnica de la estructura. Descompone sistemas complejos en Contenedores y Componentes visualizables. | *"Usa `c4-architecture` para armar un esquema de cómo el Frontend se conecta a la Base de Datos.*" |

## 3. Código Limpio y Refactorización (Deuda Técnica)

El "escuadrón de limpieza" que asegura que el proyecto nunca sea un espagueti ilegible.

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`clean-code`** | Redacta y edita código basándose en el libro 'Clean Code' de Robert C. Martin. Pone nombres legibles, divide funciones largas. | *"Aplica la skill `clean-code` a todo el archivo `ObservacionesGlobales.jsx`. Elimina 'magic numbers' y separa lógica."* |
| **`codebase-cleanup-refactor-clean`**| Exterminador masivo de deuda técnica y reestructurador. | *"Usa `codebase-cleanup-refactor-clean` destructivamente en `App.jsx` para quitar lo viejo y modernizarlo."* |
| **`debugging-strategies`** | Cuando tienes un error "imposible" o un pantallazo blanco. Usa métodos científicos de diagnóstico. | *"Tenemos un bug crítico en Vercel con las rutas dinámicas. Usa `debugging-strategies` para encontrar el origen."* |
| **`code-reviewer`** | Para revisar los cambios que estás a punto de subir, simulando un líder técnico exigente. | *"Activa `code-reviewer` en el código local de hoy. Critícalo constructivamente antes de mi Push."* |

## 4. Backend, Bases de Datos y Rendimiento

La capa oculta del software; rapidez, escalabilidad y almacenamiento.

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`database-architect`** | Diseño de relaciones (SQL/NoSQL), índices y optimización de bases de datos que no colapsen al tener 10,000 registros. | *"Desactiva el instinto frontend y activa `database-architect`. Propón un esquema SQL para Notas y Consultas."* |
| **`nodejs-backend-patterns`** | Patrones maduros en creación de APIs y servidores backend en JS/TypeScript. | *"Usa `nodejs-backend-patterns` para estructurar los middleware de seguridad en nuestras rutas."* |
| **`performance-engineer`** | Identificación de "fugas de memoria" (memory leaks) y optimización general de la app para que cargue ultra rápido. | *"Corre la skill `performance-engineer` y analízame por qué la tabla de Reportes tarda 3 segundos en renderizarse."* |
| **`sql-optimization-patterns`** | Optimización radical de análisis y consultas complejas (Queries) lentas de bases de datos. | *"Modo `sql-optimization-patterns`: Revisa este Query de filtros múltiples de fechas y dime cómo usar índices."* |

## 5. Seguridad y Pentesting Integrado

Para aplicaciones empresariales como AuditFlow Pro, tu propio equipo "Red Team" que busca hackearte antes de salir a producción.

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`security-audit`** | Revisión global post-código. Busca malas prácticas. | *"Ejecuta un `security-audit` riguroso en los inputs de los formularios. ¿Alguien podría insertar código acá?"* |
| **`threat-modeling-expert`**| Ingeniería contra ciberataques teóricos (evalúa riesgos si tus bases de datos se filtraran o interceptaran). | *"Usa la teoría STRIDE vía `threat-modeling-expert` y búscame las tres grandes amenazas de la vista Admin."* |
| **`xss-html-injection`** | Pruebas especializadas para ataques de "Cross Site Scripting" (cuando un hacker mete `<script>` por un campo de texto). | *"Usa `xss-html-injection` contra el formato donde guardan los Correlativos para ver si estamos protegidos."* |
| **`broken-authentication`** | Auditoría lógica alrededor de los mecanismos de inicio de sesión y cookies. | *"Revisa nuestra simulación de Login usando `broken-authentication`. ¿Podemos prevenir la suplantación?"* |

## 6. Testing, QA y Prevención de Errores

"Si no está probado, está roto." 

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`tdd-workflow`** | Desarrollo guiado por pruebas. Ciclo *Red-Green-Refactor*. Te fuerza a escribir pruebas antes del código lógico para cero bugs. | *"Modo `tdd-workflow`. Necesito un componente para graficar auditorías. Primero dame las pruebas de Vitest."* |
| **`playwright-skill`** | IA que puede autoprogramar un robot de navegador Chrome para probar toda tu aplicación tal como lo haría un humano. | *"Activa `playwright-skill`, abre la pestaña Inicio de sesión, llena las credenciales y toma un pantallazo si da error."* |
| **`unit-testing-test-generate`**| Escribe pruebas unitarias impecables y difíciles sobre archivos ya construidos. | *"Usa `unit-testing-test-generate` para añadirle Tests que cubran 100% el archivo principal de Reportes de React."* |

## 7. Documentación y Estandarización

No odies documentar; la IA automatizada lo hace con formatos corporativos increíbles de la noche a la mañana.

| Nombre de la Skill | Cuándo Usarla (Especialidad) | Ejemplo de Instrucción |
| :--- | :--- | :--- |
| **`wiki-architect`** | Ingiere una carpeta de tu código y saca páginas Wiki de alta calidad y diagramas *Mermaid* (diagramas como código). | *"Genera una lectura para un nuevo empleado vía `wiki-architect` explicando cómo funciona la conexión de los 4 módulos."* |
| **`api-documentation-generator`**| Escribe especificaciones estilo estándar mundial (OpenAPI/Swagger) sin margen de error, amigables para terceros. | *"Toma nuestras rutas JS locales y usa `api-documentation-generator` para hacer un Markdown muy técnico del Backend."* |
| **`git-commit-formatter`** | Un estandarizador para tu historial. Todos tus cambios en Github lucirán profesionales. | *"Redactame el mensaje para Github por favor, aplicando la disciplina del `git-commit-formatter` sobre lo hecho hoy."* |
